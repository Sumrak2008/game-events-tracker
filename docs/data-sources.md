# Реестр источников данных

Главный принцип: **нет одного универсального источника для всех игр.** Для каждой игры в `data/sources.json` хранится отдельный реестр (тип `GameSourceRegistry` в [`src/lib/types.ts`](../src/lib/types.ts)) с тремя ролями источников:

| Роль | Назначение |
| --- | --- |
| `discoverySources` | Найти максимально полный перечень текущих и анонсированных баннеров/событий/сезонов. |
| `verificationSources` | Подтвердить точные даты, регион и награды конкретной записи. Как правило, официальные. |
| `fallbackSources` | Использовать, если источники обнаружения/подтверждения недоступны или неполны. |

Официальный сайт не обязан быть единственным источником — он имеет высший приоритет среди источников **подтверждения**, но специализированные сайты разрешены и необходимы для полноты на этапе **обнаружения**.

## Структура записи источника

Каждый источник (`GameSourceEntry`) содержит:

| Поле | Тип | Описание |
| --- | --- | --- |
| `id` | string | Уникальный идентификатор источника. |
| `name` | string | Человекочитаемое название (страница/раздел сайта). |
| `baseUrl` | string | Корневой URL сайта (не обязательно точная глубокая ссылка). |
| `role` | `discovery \| verification \| fallback` | Должна совпадать с массивом, в котором находится запись. |
| `sourceType` | `official \| specialist \| wiki \| community` | Категория источника. |
| `priority` | number | Меньшее число — проверяется раньше внутри своей роли. |
| `supportsEvents` | boolean | Покрывает события. |
| `supportsBanners` | boolean | Покрывает баннеры. |
| `supportsSeasons` | boolean | Покрывает сезоны. |
| `supportsRewards` | boolean | Содержит сведения о наградах. |
| `supportsRegionalDates` | boolean | Содержит региональные/серверные даты и обратный отсчёт. |
| `enabled` | boolean | `false` — источник временно не используется (например, точный URL не подтверждён). Не брать из него данные, пока не проверено и не включено. |
| `notesRu` | string (опц.) | Примечание на русском: как использовать, ограничения, что проверить. |

Реестр игры (`GameSourceRegistry`) может также содержать `notesRu` — сквозные правила сбора для этой игры (что добавлять/не добавлять, на какие категории контента смотреть).

## Правила обнаружения, проверки и оценки достоверности

1. Открыть источник(и) обнаружения для игры.
2. Составить полный список текущих и анонсированных баннеров, событий и сезонов.
3. Открыть отдельную страницу каждой записи — поисковый сниппет источником не считается.
4. Извлечь даты, регион, награды и требования.
5. Попытаться подтвердить запись источником подтверждения (официальным, если есть) → `confirmed`.
6. Если официального подтверждения нет, но совпадают минимум два независимых специализированных источника → `corroborated`.
7. Если найден только один приемлемый источник → `single-source`.
8. Если источники противоречат друг другу → `conflicting` (с объяснением в `verificationNote`, без произвольного выбора одной даты).
9. Если сведения найдены, но проверить их пока нельзя ни одним источником → `unverified`.
10. Сохранить прямые URL всех использованных страниц в `sourceUrls`.
11. Не прекращать поиск после проверки одного официального сайта — пройти всю цепочку обнаружение → подтверждение → резерв.

Запрещены как подтверждение: фан-вики без указания источников, Reddit, форумы, агрегаторы, непроверенные списки, страницы с утечками и прогнозами будущих версий.

## Что нужно собирать

Добавлять все временные внутриигровые события, которые дают хотя бы одно из следующего: премиальную валюту, валюту призыва, билеты/крутки, бесплатного персонажа, оружие/экипировку, костюм, уникальную ограниченную награду, значимую валюту события. Также: события входа, пробные забеги, временные боевые режимы, циклические эндгейм-режимы с обновляемыми наградами, веб-события с внутриигровыми наградами. **Не добавлять постоянный контент без временного окна.**
