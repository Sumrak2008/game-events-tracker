# Game Events Tracker — правила работы

Трекер игровых **баннеров, событий и сезонов**. Данные хранятся в JSON-файлах в `data/` и являются единственным источником правды. Базы данных и админ-панели нет — данные меняются правкой JSON (в т.ч. через Claude Code) с последующим пушем в GitHub и деплоем на Vercel.

## Архитектура (кратко)

- `data/*.json` — игры, баннеры, события, сезоны, источники.
- `src/lib/` — типы (`types.ts`), загрузка данных (`data.ts`), вычисление статуса (`status.ts`), форматирование (`format.ts`), подписи (`labels.ts`).
- `src/components/` — UI. `src/app/` — страницы (App Router).
- `scripts/validate-data.mjs` — валидатор данных (`npm run validate`).
- Подробности формата — в [docs/json-format.md](docs/json-format.md).

## Модель данных (обязательно соблюдать)

- Типы записей строго: `banner`, `event`, `season`. Ничего больше (без промокодов, патчей, техработ, новостей, рейдов, Twitch Drops и т.п.).
- Обязательные поля: `id`, `type`, `gameId`, `title`, `description`, `startAt`, `endAt`, `timezone`, `region`, `sourceType`, `confidence`, `sourceUrls`, `verifiedAt`, `isDemo`.
- Опционально: `verificationNote`, `imageUrl`, `note`, `sourceUrl` (устар., вместо него `sourceUrls`).
- Для баннеров опционально `bannerSubtype` (`character`/`weapon`/`equipment`/`other`).
- Для событий опционально: `eventSubtype` (`major`/`challenge`/`login`/`web`/`limited-mode`/`reward-event`/`other`), `rewards` (массив строк), `rewardSummary`, `premiumCurrencyName`, `premiumCurrencyAmount`, `summonCurrencyAmount`, `requirements`, `claimEndAt`.
- Даты — ISO 8601 **с указанием часового пояса** (например `2026-06-19T22:00:00+08:00`).
- Статус (`upcoming`/`active`/`completed`) **не хранится** — он вычисляется по текущему времени, `startAt` и `endAt`. Никогда не добавляй поле статуса в JSON.

## Источники: приоритет (от высшего к низшему)

1. Официальные сайты игры, разработчика, издателя.
2. Официальные внутриигровые объявления, новости, календари.
3. Официальные соцсети и официальные сообщества.
4. Крупные специализированные игровые сайты и трекеры.
5. Поддерживаемые вики **с указанием источников**.
6. Надёжные сообщества/базы данных — только если подтверждаются другими источниками.

**Официальные источники имеют высший приоритет, но поиск ими не ограничивается.** Поисковые сниппеты источником НЕ являются — всегда открывай исходную страницу. Запрещены как подтверждение: фан-вики без источников, Reddit, форумы, агрегаторы, непроверенные списки.

## Достоверность (`sourceType` + `confidence`)

- `sourceType`: `official | specialist | wiki | community` — категория самого сильного источника.
- `confidence`:
  - `confirmed` — дата подтверждена **официальным** источником (требует `sourceType: official`).
  - `corroborated` — дата совпадает минимум в **двух независимых** сторонних источниках.
  - `single-source` — найден только один приемлемый сторонний источник (пониженная уверенность).
  - `conflicting` — источники дают разные даты/условия.
- `sourceUrls` — массив **всех** использованных ссылок (первая — сильнейшая).
- `verificationNote` — причина неопределённости / как сверялось.

## Правила работы с данными (СТРОГО)

1. **Приоритет официальных источников**, но не ограничивайся ими — собирай максимально полную информацию по цепочке приоритета выше.
2. **Не придумывай даты** и не выдавай предположение за факт. Точное количество наград не выдумывай: если подтверждён только факт премиальной валюты — укажи `premiumCurrencyName` без количества.
3. **Сторонние данные перепроверяй:** ищи минимум два независимых источника; сравнивай название, даты начала/окончания, регион/сервер; сохраняй все ссылки в `sourceUrls`.
4. **При конфликте дат** не выбирай одну произвольно — ставь `confidence: conflicting`, объясняй в `verificationNote`. Такие записи в UI идут в блок «Требует проверки», не среди подтверждённых.
5. **Учитывай регион и часовой пояс**: заполняй `region` и `timezone`, даты — с корректным смещением.
6. **Не удаляй завершённые записи.** Архив формируется автоматически по прошедшему `endAt`.
7. **Не заменяй подтверждённое неподтверждённым.** `confirmed`/`corroborated` нельзя ухудшать до `single-source`/догадки.
8. **`isDemo: true`** для демонстрационных и ориентировочных записей (в UI — пометка «Демо-данные»). Не выдавай вымышленное за реальное.
9. **Изображения с сайтов игр не копируй.** Используется оригинальный SVG-арт (`public/art/games/<id>.svg`); `imageUrl` — задел под собственные растровые изображения.
10. **Полнота:** не останавливайся, если на офиц. сайте нет данных — пройди всю цепочку источников. Собирай не только баннеры, но и временные события с наградами (премиум-валюта, билеты призыва, персонажи и т.п.) и сезоны. Постоянный контент без даты окончания не добавляй.
11. **Перед публикацией показывай отчёт об изменениях** и жди подтверждения.
12. **Не публикуй при ошибках проверок** (`validate`/`lint`/`typecheck`/`build`).

## Проверки перед публикацией

```bash
npm run format      # Prettier
npm run lint        # ESLint
npm run typecheck   # tsc --noEmit
npm run validate    # валидация JSON
npm run build       # production-сборка
```

Все пять должны проходить. Подробности публикации — в [docs/publishing.md](docs/publishing.md).
