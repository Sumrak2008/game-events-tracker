# Game Events Tracker

Веб-приложение для отслеживания **игровых баннеров, событий и сезонов** по нескольким играм. Это не gacha-only трекер — в списке есть как gacha-проекты (Genshin Impact, ZZZ, Honkai: Star Rail, Arknights: Endfield, Neverness to Everness, NIKKE), так и Diablo IV и World of Warcraft.

Все данные хранятся в **JSON-файлах внутри репозитория** (`data/`). После пуша в GitHub и деплоя на Vercel обновления видны на всех устройствах. Отдельной базы данных и админ-панели нет — данные обновляются правкой JSON (в т.ч. через Claude Code).

## Стек

- **Next.js 16** (App Router) + **React 19** + **TypeScript**
- **Tailwind CSS v4**
- ESLint + Prettier
- Node.js-скрипт валидации данных (без зависимостей)

## Возможности

- Главная со сводкой: заканчивается сегодня / в ближайшие 3 дня, активные баннеры, события, сезоны, ближайшие предстоящие, дата последней проверки данных.
- Список игр и отдельная страница каждой игры.
- «Скоро закончатся» — активные записи, отсортированные по ближайшему окончанию.
- Календарь начала/окончания записей.
- Архив завершённых записей (формируется автоматически по `endAt`, ничего не удаляется).
- Страница деталей записи.
- Фильтры: игра, тип, статус, регион. Сортировка по ближайшему окончанию.
- Статус (`upcoming` / `active` / `completed`) **не хранится**, а вычисляется по текущему времени.

## Типы записей

Только три: `banner`, `event`, `season`. Никаких промокодов, патчей, технических работ, новостей, рейдов, Twitch Drops и прочих категорий.

## Структура

```
data/                 JSON-данные (источник правды)
  games.json
  banners.json
  events.json
  seasons.json
  sources.json
scripts/
  validate-data.mjs   валидатор данных
src/
  app/                страницы (App Router)
  components/         UI-компоненты
  lib/                типы, загрузка данных, вычисление статуса, форматирование
docs/                 документация формата и инструкции
```

## Разработка

```bash
npm install      # установка зависимостей
npm run dev      # дев-сервер (http://localhost:3000)
```

## Проверки перед публикацией

```bash
npm run format       # форматирование Prettier
npm run lint         # ESLint
npm run typecheck    # проверка типов TypeScript
npm run validate     # валидация JSON-данных
npm run build        # production-сборка
```

Агрегат проверок без сборки: `npm run check`.

## Данные и достоверность

Демонстрационные записи помечены `"isDemo": true` и в интерфейсе подписаны «Демо-данные»; их даты вымышлены. Реальные данные добавляются с сохранением ссылок (`sourceUrls`), типа источника (`sourceType`), уровня достоверности (`confidence`) и даты проверки (`verifiedAt`). Подробнее — в [CLAUDE.md](CLAUDE.md).
